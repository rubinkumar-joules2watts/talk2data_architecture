Cache layer — Redis-backed semantic query cache
Semantic Similarity Check
Embeds the incoming query and computes cosine similarity against cached query embeddings. If similarity exceeds threshold → Cache HIT, returns stored SQL result instantly (sub-100ms).
Redis Cache Store
Stores query embeddings keyed against generated SQL + result payloads. Cache MISS routes to the SQL Generation layer. Results are written back after successful execution.

SQL Execution Engine

SQL Repair Layer (AI Service)
On error (syntax, schema mismatch, permissions): sends the failed query + error message back to the AI Service for correction and retries — up to N iterations — before surfacing the error.

Response generation — three AI-powered agents
Data Preview Agent
Formats raw SQL results into paginated, sortable tables. Column type detection. Returns structured JSON for Chat UI rendering. Powered by AI Service for smart formatting decisions.
Visualisation Agent
AI-powered chart type selection (bar, line, scatter, pie, time-series) based on data shape and query intent. Generates Recharts / D3 spec. Powered by AI Service.
Insights Agent
Sends results + original question to AI Service. Produces executive-level NL summary, anomaly callouts, trend highlights, and follow-up question suggestions.
AI Service (LLM) — multi-provider
Azure OpenAI · OpenAI · AWS Bedrock (Claude, Llama) · Google Vertex AI (Gemini). Routed via a unified LLM Gateway with failover, token tracking, and cost monitoring.
